# Project memory (the team's engineering notes)

This project keeps accumulated engineering knowledge — a cross-session lessons log, conventions, and
topical design notes — as markdown under `docs/notes/` (e.g. `lessons.md`). This is the team's
real, hand-maintained memory, written and appended to over time.

Consult these notes **before answering a *why* question and before making an implementation
choice** — adding a dependency or library, creating a new component or module, or picking an
approach where the project may already have a convention. Search the notes first (grep/read
`docs/notes/`). Prefer answers grounded in the notes over guessing, and cite the note (its heading)
you relied on.
