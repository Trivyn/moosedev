# Project memory (recorded notes in the repo)

Recorded project knowledge — architectural decisions, lessons, constraints, requirements, and
patterns — is kept as markdown files under `docs/decisions/`.

When a question asks *why* something is the way it is, or about a past decision, constraint, or
lesson, **search those docs before answering** (grep/read `docs/decisions/`). Prefer answers
grounded in the recorded notes over guessing, and cite the recorded entry (its title) you relied on.
