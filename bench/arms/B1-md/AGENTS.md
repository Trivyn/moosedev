# Project memory (recorded notes in the repo)

This project has recorded knowledge — architectural decisions, lessons, constraints, requirements,
and patterns — kept as markdown files under `docs/decisions/`.

Consult project memory **before answering a *why* question and before making an implementation
choice** — adding a dependency or library, creating a new component or module, or picking an
approach where the project may already have a convention. Search the recorded notes first
(grep/read `docs/decisions/`). Prefer choices and answers grounded in the recorded notes over
guessing, and cite the recorded entry (its title) you relied on.
